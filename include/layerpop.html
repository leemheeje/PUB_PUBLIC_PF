신규&운영 프로젝트를 처음 받게 될 때면, UI 개발, UI 고도화, UI 개선 등등 UI/UX 관련 작업을 많이 하게 될때면,
"조금 더 간결화 시키며, 조금 더 접근하기 쉽고, 유지 보수가 용이할까?"
라는 고민을 합니다. 그러다 보편적으로 사용하는 웹페이지의 UI는 자신만의 스타일로 정의 해놓는 것이 굉장히 편할 것이라 생각했습니다. 물론, jQueryUI, Semantic, BootStrap 등등 글로벌 라이브러리를 사용할 순 있지만, UI 개선 및 고도화 작업이 이루어질 때면(예:라이브러리를 커스터마이징 해야 할 때) 알 수 없는 문제점을 발생시킬 수 있습니다.
  1. 1. 물리적 시간 투자
  2. 2. 해당 라이브러리를 완벽한 이해
  3. 3. 라이브러리에 없는 신규 기능 추가
  4. 4. 프로젝트 특성상 라이브러리 자체를 사용할 수 없을 때
물론 위 문제점을 모두 완벽하게 해결할 순 없지만 제한된 시간 안에 일을 해야 하기에 보편적으로 사용하는 UI구성을 모두 내장시켜 버전업하며 사용하고 있습니다.
